Github

Source Code

A Github page could contain the source code for Web Apps or locally run scripts (which can help with PrivEsc / Post-Exploitation).

Useful Info

Creds: Hardcoded Usernames &/or Passwords

Subdir Enum: Web App Directory Structure. However, Repo may not be used in Web Root (e.g. /wordpress/ ) & it may not have ALL the files from the Github Repo. Check for multiple known files to identify parent directory.

Back-End Code: Viewing the BE Code can help to identify dependencies, gadgets for exploitation, etc.

Commit History

Sensitive info may have been removed, but the Commit History is a permanent record of changes made to a Repo. Look for lines that were deleted.

Note: Sensitive data may have also been added accidentally, so also check lines that were added. Potentially less likely than finding it in a deleted line though.

Github Account

[CTF2 106]

Even if the source code isn't available for a company's Github Account, you could find other userful info (e.g. users which could be used for a Local / Domain Account, Web App, or other service like FTP)

Click an Account & examine the Repositories & People listed.

Note: An account listed under People may also have Repos utilized by the target Company.

DB Breach

Search for various info like Email Adresses on leakpeak.com to discover leaked passwords & hashes (only get partials if using free version).

ProxyNova gives you full cleat-text data, but only checks a 2021 dataset called COMB (Combo of Many Breaches)

Get Full Password

You'll know the length of the password, so you can try guessing common permutations & use crunch to generate the rest. For example, to guess the last character in the password-

crunch 13 13 -t XamplePasswd% >> crunch.txt

crunch 13 13 -t XamplePasswd^ >> crunch.txt

Reverse Image Search

TinEye can lookup images (e.g. a company's homepage to see if a Web Dev listed in in their portfolio. The Web Dev's email may be a username for their client's site).
