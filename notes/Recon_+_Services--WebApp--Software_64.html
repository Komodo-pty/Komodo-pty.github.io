Boundary (multipart) Data

[CTF B 190-191]

One method for processing data in a POST Request is to use an HTTP Header that specifes multipart data.

Content-Type: multipart/form-data; boundary=--STRING

The string can be any sequence of any ASCII printable characters, but no spaces or special characters. The string needs to start with at least 2 dashes --.

Closing the Body

The end of the Request's body data will contain the boundary with 2 dashes (--) appended to it.

Boundary Use Case

The boundary acts as a delimiter, marking the start & end of each part of the Request body when sending multipart data.

Each part of the body represents a different piece of data (e.g. from a file upload or form fields). The boundary seperates each of these parts so that the server can correctly parse the individual data elements.

Identifying Uploaded File Type

When multipart data is used for file uploads, there will be the normal HTTP Header which specifies the boundary (shown aboce), but there will also be ANOTHER Content-Type contained within a field of the body data.

Note: The piece of body data that identifies the file type (MIME Type) can sometimes be spoofed to bypass filters without breaking your payload.
